The Bumps, glitches and issues in the V20 Roll-out

All the issues with the new Gen of CU!

 

 

So there is no denying that the V20 rollout was less than ideal, in the history of rollouts V20 was possibly the best update we have ever received but at the same time the most flawed with issues rendering images, speed and bugs all over the place. But some might wonder why this was the case, and what work was went into bringing the V20.0.0 update to everybody in the stable way it is today.

 

To get the best idea into just how much work went into this we have to roll back to the Beta 1 release, Yes that far back. At the launch of V20 Beta 1 we saw L-NET take hold of the new website and host it to the public during this time we worked hard to find the best solution to host the new generation and L-NET was our focus. With L-NET though we ran into issues such as HTTPS, the web link and much more which is why today we have hosted the website on another service and kept L-NET around as the “Development Server”

Around the Beta 3 release we figured L-NET would not work for the final release so we took to the Github ran service “Github Pages” and for a long time it worked as well as it could. Running everything perfectly and hosting great even with SSL or HTTPS, We were even able to update outside of the network which was great. But come Beta 6 we noticed an issue. It was around this time we launched HDIP2x and that was a big jump in the content that we had to host. While L-NET ran fine github ran into a directory issue being unable to host the coming to 9000 items we had in 1 directory.

We made the mistake of ignoring this, Come with Beta 7 we noticed issues with many of the images not displaying properly, again we ignored this and continued with progress and come the V19 shutdown prep for the launch on github we moved Beta 8 to the L-NET dev server and hosted V19 on the github. This was fine as V19 was under the capacity but come the launch of V20 as we know it on the 2nd November 2022 we ran into major issues. Github was not displaying images, broken links and lost assets. Everything broke down.

During this time we made efforts to fix it, but to no avail github pages was unable to host the hidden content.

We was dead in the water. During the issue time we moved people to the ddns.net / development server and focused on solving issues. We went to AWS and Google Cloud but none of it worked. Only today on the 6th November, 4 days after launch we found a new service that uses github to find content and host it all not just visible stuff meaning no issues would happen.

This was thanks to Netlify.

Today we continue to use Netlify to host the website with SSL and out domain name. Runs well and much more. Welcome to V20, we did finally get here!
